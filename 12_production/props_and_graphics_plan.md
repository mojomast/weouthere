## Planning Principle

On `WE OUT HERE`, props and graphics are often the same storytelling object. A laptop sticker, investor deck, faux contract, printed agenda, browser window, pitch leave-behind, or notification screen can all carry plot.

They must therefore be managed as one coordinated system.

## Asset Categories

| Category | Examples | Lead Owner |
| --- | --- | --- |
| Hand props | phones, laptops, chargers, notebooks, bags, adapters | props |
| Paper props | contracts, invoices, notes, legalese, printed decks | graphics with props handling |
| Environmental graphics | signage, labels, desk tags, whiteboards, packaging | art/graphics |
| Story graphics | pitch decks, dashboards, screenshots, timelines | graphics with post visibility |
| Screen-linked physicals | printed QR cards, badges, sticky notes matching UI | props + graphics |

## Hero Prop List Priorities

The first pass should identify every recurring hero item in these buckets:

- lead character phones
- laptops and peripherals
- bags and wearable tech accessories
- printed business materials for Cataclysm
- legal/compliance paperwork used for comedy and plot
- presentation objects such as clickers, adapters, dongles, cheap swag, and branded leave-behinds

## Graphics Philosophy

- everything should feel plausible enough to be mistaken for real contemporary work product
- avoid parody that reads as sketch-comedy fake unless a scene explicitly wants that register
- graphics must look legible in close-up and believable in peripheral view
- legal and brand clearance concerns should be solved upstream, not painted out later

## Approval Chain

| Asset Type | Drafted By | Reviewed By | Final Approval |
| --- | --- | --- | --- |
| Printed paperwork | graphics | props, script, legal if sensitive | production_agent |
| Branded Cataclysm materials | graphics | art, showrunner if prominent | production_agent |
| Screen-matching physicals | graphics | VFX/screens lead, props | production_agent + post consult |
| Hero devices | props | camera, sound, post if screen-linked | production_agent |

- digital source package delivered to post-facing storage

## Continuity And Version Control

- every recurring graphic gets a version number
- every printed hero prop gets a source file retained in organized storage
- if a screen asset and a paper asset represent the same document, both must reference the same version state
- script supervision should log which version appears in take when story-critical

## Practical Device Plan

- standardize device families where possible for charging, accessories, and replacement simplicity
- preconfigure hero devices before set use
- lock wallpapers, notifications, and visible naming conventions before first use
- no personal crew devices on camera unless formally cleared into continuity

## Risk Controls

- do not let day-of props creation become a substitute for prep
- do not introduce unapproved logos or real company marks into background clutter
- do not assume unreadable printed props will never end up in close-up
- do not let multiple departments output different versions of the same story document
