## Workflow Goal

The show needs one production operating system. Departments can have specialized practices, but the project should behave as one coordinated pipeline from prep through post turnover.

## Communication Workflow

### Daily Production Chain

| Function | Primary | Secondary |
| --- | --- | --- |
| schedule execution | 1st AD | line producer |
| budget / overtime / spend pressure | line producer | production_agent |
| creative interpretation on set | director / showrunner | script supervisor |
| graphics / screen approvals | production_agent | post_agent |
| media and turnover integrity | DIT / data manager | editor / AE contact |

### Rule

All cross-department decisions that affect schedule, graphics, VFX, or editorial must be logged through production coordination the same day.

## Prep Workflow

1. Scripts broken down into production categories.
2. Art, props, graphics, camera, sound, and post identify hybrid-format needs.
3. AD and line producer issue stripboard and budget alignment pass.
4. Pipeline test is run for screens, sound, and data.
5. Final workflow packet issued before tech scout.

## On-Set Workflow

### Start Of Day

- confirm call sheet updates
- confirm special screen or playback needs
- confirm sound risks and room tone plan
- confirm data manager knows any non-standard capture types for the day

### During Day

- script supervisor logs takes, notable improv, and graphics/screen versions used
- AD flags schedule drift by lunch, not at wrap
- any screen discrepancy triggers immediate reference capture and note to post
- any prop/graphic continuity change is photographed and logged

### End Of Day

- media backed up and verified
- camera/sound/screen reports consolidated
- editor turnover package created
- unresolved continuity or pickup items logged before crew release if possible

## Screen Asset Workflow

1. Request originates from script, art, or production.
2. Graphics/UI build prepared and versioned.
3. Production approves use case: practical playback, native capture, or both.
4. Asset is saved into approved folder structure.
5. On set or capture day, exact version used is logged.
6. Editorial receives both used and clean source where applicable.

## Change Management

### Minor Change

- typo fix
- cosmetic graphics tweak
- non-story prop replacement

Handled by department lead with coordination notice.

### Major Change

- new location
- new screen sequence
- new surreal beat
- wardrobe change affecting continuity
- playback change affecting performance

Requires same-day notice to AD, line producer, script supervision, and post-facing contact.

## File And Naming Workflow

- all scene-critical assets need consistent naming and versioning
- no final assets should live only on an individual's local desktop
- screen and graphics files must distinguish clean, playback, reference, and final-approved states
- editorial-facing folders should match turnover language used by production reports

## Turnover Workflow To Editorial

Daily turnover should contain:

- camera media reports
- sound reports
- screen asset index
- notes on playback and source versions
- continuity stills when relevant
- outstanding issues list

## Contingency Workflow

### If Location Fails

1. AD and line producer move to cover set or pre-approved alternate.
2. production_agent logs affected scenes.
3. post and graphics are notified if any screen or art continuity is affected.

### If Screen Workflow Fails

1. capture reference and clean plate immediately
2. log exact issue
3. preserve performance if possible
4. route replacement plan before end of day

### If Schedule Slips

1. AD issues realistic remaining-day plan
2. line producer assesses cost and next-day impact
3. director/showrunner prioritize story-critical material
4. pickups are logged with category and urgency, not just as a loose list

## Production Reports

The production office should issue:

- daily production report
- hot-cost view during shoot weeks
- pickup list by priority
- open graphics / screen request list
- unresolved continuity issue log

## Workflow Success Metrics

- editor receives coherent dailies and screen assets every day
- no department asks post to guess what version was used
- continuity issues are identified during production, not after picture assembly
- schedule recovery choices protect story-critical material first
