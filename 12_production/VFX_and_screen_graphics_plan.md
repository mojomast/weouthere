## Scope Principle

The VFX plan is intentionally narrow. `WE OUT HERE` should rely on:

- practical screen capture
- controlled monitor playback
- designed graphics packages
- selective invisible support VFX
- specific surreal sequences with clear narrative purpose

It should not rely on broad post rescue.

## VFX Categories

| Category | Typical Use | Default Handling |
| --- | --- | --- |
| Screen replacement / cleanup | monitor flicker, glare, version mismatch | only when practical capture cannot carry final frame |
| Interface enhancement | timing polish, readability, extension | based on approved source builds |
| Invisible cleanup | crew removal, boom cleanup, wall patch, signage cleanup | case by case, controlled |
| Surreal inserts | psychologically motivated visual ruptures | pre-boarded, separately estimated |
| Graphics composites | calls, layered windows, data overlays | built from approved asset packages |

## Screen Graphics Rules

### Core Rules

1. The fictional platform and all in-world interfaces must be designed before shoot use.
2. Any screen asset used on set must have a retained source package.
3. Editorial receives both practical-capture versions and clean source versions.
4. Post does not redesign story content after the fact unless formally approved as a change order.

### Interface Package Structure

Each recurring interface family should include:

- master design file
- font and asset package
- state variants
- clean plate exports
- playback exports if needed
- editorial reference movie or stills

## Surreal Insert Policy

Surreal material is part of the show's identity, but it is a controlled accent, not a second genre.

A surreal sequence must have all of the following before it enters the schedule:

- written story purpose
- visual concept
- practical capture plan
- VFX scope note
- editorial and sound implication note

If one of those is missing, the sequence is deferred, simplified, or cut.

## Turnover Requirements

For every VFX or screen-graphics shot, turnover should include:

- shot ID
- source plate(s)
- reference export
- script/context note
- frame range or editorial locator when available
- intended final behavior
- asset package path
- approval status

## Approval Chain

| Decision | Primary Owner | Consult |
| --- | --- | --- |
| interface design lock | production_agent | brand_agent, post_agent |
| practical playback build approval | production_agent | DP, editor |
| VFX shot inclusion | post_agent | showrunner_agent, production_agent |
| surreal sequence expansion | showrunner_agent | line producer, post_agent, DP |
| post-only rescue work beyond plan | post_agent | line producer, EP |

## Budget Protection Rules

- every surreal sequence is estimated separately
- invisible cleanup should be tracked so it does not silently become a second workload
- screen replacements should be the exception, not the baseline expectation
- no interface family should multiply into many versions without change tracking

## Common Failure Modes To Prevent

- untracked screen version seen on set but not saved centrally
- graphics built in styles that camera cannot photograph cleanly
- editorial cutting with temp screens that diverge from final approved source
- VFX learning too late that a practical monitor angle is unusable
- surreal sequence scope drifting because no one defined the actual shot count

## Required Early Test

Before principal photography, run one full-chain test:

- build interface asset
- play it practically on set
- capture it in camera
- capture native screen record
- turn over both to editorial
- test one replacement/enhancement path in post

If this test is weak, the production should fix the pipeline before shooting additional screen-heavy material.
