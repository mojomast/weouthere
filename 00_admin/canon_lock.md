## Purpose & Authority

This document is the **single source of truth** for the television project WE OUT HERE.

Every deliverable, script, marketing asset, legal document, press release, and internal memo produced for this project must conform to the values defined here. If any document anywhere in the project contradicts this file, **that document is wrong** — not this one.

`canon_lock.md` supersedes `studio_manifest.md`, `series_bible.md`, `character_bible.md`, `tone_guide.md`, and all other project documents on any point where they conflict. Those documents may elaborate on canon, but they may not contradict it.

No agent may override, reinterpret, or "creatively adjust" the values in §1 without following the Amendment Protocol in §6.

### 1.4 Tone

Canon tone is defined by the following attributes, all of which must be present:

- **Smart, internet-literate, psychologically sharp, humane, and darkly funny**
- **Absurdist but emotionally grounded**
- **Discord realism, startup satire, labor anxiety, failed intimacy, legal comedy, meme surrealism**

**Comp frame:** Prestige indie sitcom + startup collapse comedy + psychoanalytic group chat theater.

### 1.5 Season Arc

The six-beat season arc, in exact order:

> **Mania → fragile success → ethical drift → containment failure → ironic collapse → exposure to consequences**

Each beat maps to one episode. The arc is a one-way ratchet — the show does not reset to stability between episodes. Instability accumulates.

### 1.8 Writerly Constraints (Canon-Level)

These are not suggestions. They are load-bearing rules that define what the show is.

1. **Comedy must emerge from defense mechanisms, labor anxiety, and failed containment.** Not from wacky misunderstandings, not from characters being stupid, not from sitcom contrivance. The humor is psychological.

2. **Chat UI and screenlife components must be producible, not novelistic.** If you can't shoot it as a real screen recording, redesign it. No "the chat scrolls through 200 messages while orchestral music plays" unless you can actually build and film that interface.

3. **No moralizing speeches that dissolve the comedy.** Characters may have insights, but the show does not pause to lecture the audience. If a character delivers a monologue about What It All Means, the show must undercut it — either immediately or within the scene.

4. **Every episode must have at least one emotionally true moment hidden inside a joke sequence.** The audience should feel something real in every episode, but the show should never announce that it's being sincere.

5. **Endings preserve instability rather than resolve cleanly.** No episode ends with the group having Learned A Lesson. Problems compound. Partial wins create new problems. The season finale is not redemption — it is exposure to consequences.

6. **The trio's psychodynamic logic must be preserved at all times.** twinkle.puss is not "the fun one." bl0ng is not "the smart one." ariane.emory is not "the weird one." They are defined by their psychological defense structures, and those structures must drive their behavior in every scene.

| 11 | **Soft locks respected?** | Soft lock values from §2 are used unless a documented change has been approved. |

## §5 — Canon vs. Elaboration

Not everything in the project is canon-locked. This section clarifies the boundary.

### 5.1 What Is Canon (defined here, immutable)

- Show title, format, genre, production style
- Logline (verbatim)
- Character handles, archetypes, and psychological profiles
- Season arc (six beats, exact order)
- Episode titles (exact text, exact order)
- Tone definition and comp frame
- Writerly constraints (all six)
- Platform rule (Discord-like, never literally Discord)

### 5.2 What Is Elaboration (defined in other documents, must not contradict canon)

- Character backstories, relationships, specific dialogue patterns → `character_bible.md`
- World-building details, setting specifics → `world_bible.md`
- Episode plots, scene breakdowns, B-stories → `episode_outlines.md`, story breaks
- Tone examples, do/don't lists → `tone_guide.md`
- Visual style, color, typography → `brand_guide.md`
- All scripts, marketing materials, legal documents

### 5.3 What Is Operational (not canon, governed by studio_manifest.md)

- Agent assignments, wave schedules, approval chains
- File naming conventions, document formatting
- Communication protocols, conflict resolution
- Production logistics, crew plans, budgets

**Rule:** Elaboration documents may add detail but never contradict §1. Operational documents are governed by `studio_manifest.md` and do not require canon review unless they reference creative content.

## §6 — Amendment Protocol

Canon is designed to be stable. Changing it should be rare, deliberate, and documented.

### 6.1 Who Can Propose Amendments

Any agent may propose an amendment, but amendments are only approved by:

- **Immutable canon (§1):** Requires showrunner approval **AND** ops_agent sign-off. Both must agree.
- **Soft locks (§2):** Requires the approver(s) specified in the §2 table for that item.

### 6.2 Amendment Process

1. **Proposer** writes a canon amendment request including:
   - Which section/value is affected
   - Current canon value
   - Proposed new value
   - Rationale for the change
   - Impact assessment: list every deliverable that would need to be updated
2. **ops_agent** logs the request and routes it to the showrunner.
3. **continuity_agent** assesses downstream impact across all existing deliverables.
4. **Showrunner** approves or rejects.
5. If approved:
   - ops_agent updates this document with the new value.
   - A new entry is added to the Version History (§7).
   - All affected deliverables are flagged for revision.
   - The old value is preserved in the Version History for auditability.

### 6.3 What Cannot Be Amended

Some values are considered foundational and should not be amended under normal circumstances. If any of these need to change, it signals a fundamental creative pivot that requires a full project reassessment:

- The number of main characters (three)
- The show title (WE OUT HERE)
- The genre (single-camera dark sitcom / tech psychocomedy)
- The number of episodes in Season 1 (six)

Amendments to these values require unanimous principal agreement and a documented reassessment of all project deliverables.
